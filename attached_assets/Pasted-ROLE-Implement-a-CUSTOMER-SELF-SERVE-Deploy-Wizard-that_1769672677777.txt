### 1) DEPLOY WIZARD UI
Route: /dashboard/deploy

Steps:
A) Choose Provider (REPLIT | RENDER | RAILWAY | FLY | DOCKER)
B) Enter LIVE App URL (https://...)
C) Database (DATABASE_URL)
D) Secrets (auto-generate JWT_SECRET + ENCRYPTION_KEY)
E) Optional keys (OPENAI_API_KEY, Email, Payments)
F) Pre-Flight Check
G) Verify LIVE URL
H) Mark LIVE

UI Requirements:
- Clear explanations per step (what/why/how)
- Copy buttons for secrets (shown once)
- Block ‚ÄúGo Live‚Äù if pre-flight fails
- Progress indicator

================================

### 2) ENV & SECRET AUTO-GENERATOR
API:
- POST /api/env/generate
Returns (shown once):
- JWT_SECRET
- ENCRYPTION_KEY

Rules:
- Generate server-side
- Store encrypted references only
- Warn user to save securely

================================

### 3) PRE-FLIGHT CHECK (BLOCK BAD DEPLOYS)
API:
- POST /api/deploy/preflight

Checks:
- DATABASE_URL reachable
- Required env present
- At least one admin user
- No critical module misconfig
- Setup checklist required items complete

On failure:
- Block deploy
- Human-readable fix guidance

================================

### 4) LIVE VERIFICATION
API:
- POST /api/deploy/verify

Actions:
- Call {appUrl}/health
- Call {appUrl}/ready
- Call {appUrl}/api/dashboard/stats (auth)
- Record results

================================

- DeployRun:
  id, tenantId, deployConfigId, status(passed|failed),
  resultsJson, errorText, startedAt, finishedAt

Audit Logs:
